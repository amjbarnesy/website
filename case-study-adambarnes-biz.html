This Site by Adam

Built to prove a point. And to earn it. adambarnes.biz is a bespoke, hand-coded website with no CMS, no framework, and no compromise. It exists to demonstrate exactly what becomes possible when a site is built properly from the ground up.

Project adambarnes.biz
Live site adambarnes.biz
Built with HTML · CSS · JavaScript · GSAP
Optimised for Performance · SEO · GEO

The brief

A portfolio site for a web designer and photographer has one job above all others: it has to be good. Not just presentable. Not just functional. Actually good, because the site itself is the pitch.

WordPress would have been the obvious choice. It is, after all, the platform used for most client work. But building a personal portfolio in WordPress and Elementor would have been self-undermining. It would say: I know how to use a template builder. That is not the message.

The brief, set by and for the same person, was simple. Build something that demonstrates genuine technical depth. Something that earns its credibility rather than claiming it. A site that a client with high standards would find and immediately understand what they were looking at.

The approach

No content management system. No framework. No build tools, no bundler, no dependencies beyond GSAP for animation. Just HTML, CSS, and JavaScript: written by hand, structured with care, and served as static files directly to the browser.

Every page was built with full control over the output. No theme generating bloated markup. No plugin adding unnecessary scripts. No database query on every load. The result is a site that does exactly what it needs to and nothing it does not.

A unified CSS architecture handles the full design system: fluid typography using clamp(), a CSS custom property colour system with per-page accent colours, a dual light and dark theme stored in localStorage, and a mobile-first responsive layout that adapts cleanly at every breakpoint.

The JavaScript is lean and purposeful: page transitions, scroll-triggered reveals, a custom cursor with damped following, and a hero word cycle that brings the homepage to life without ever becoming an obstacle to the content.

Performance

Without a CMS, there is nothing to slow the site down. No PHP executing on every request. No database query assembling the page. No plugin stack adding weight. The HTML is ready the moment it is requested.

Images are sized appropriately. Scripts are deferred. Animations respect prefers-reduced-motion. The custom cursor does not load on touch devices. Every decision was made with the understanding that speed is not a nice-to-have; it is part of the experience.

A fast site is a better site. It signals care and competence before a single word is read. For a designer's portfolio, that first impression matters enormously.

GEO: Generative Engine Optimization

Search is changing. Traditional SEO (keywords, backlinks, crawlability) remains important, but the way people discover information is shifting. AI-powered tools including Perplexity, ChatGPT Search, Google AI Overviews, and Bing Copilot are increasingly the first point of contact between a question and an answer.

Generative Engine Optimization (GEO) is the practice of building and structuring content so that these AI systems can find it, understand it, and cite it accurately. It is a meaningful and growing discipline, and a hand-coded site has a significant natural advantage.

AI language models read clean, semantic HTML. They parse structured data directly. They favour pages that load quickly and present information clearly, without requiring JavaScript to render the core content. A WordPress site buried under plugin output, theme wrappers, and dynamically injected markup is significantly harder for a generative engine to interpret correctly. A clean, well-structured HTML page is not.

This site was built with that in mind from the start:

  • Every page carries full JSON-LD structured data (Person, Service, CreativeWork, FAQPage, BreadcrumbList), giving AI systems clear, unambiguous signals about what the page is, who it belongs to, and what it describes
  • Semantic HTML throughout: proper heading hierarchy, landmark elements, descriptive labels, and meaningful structure that a machine can follow as naturally as a human
  • The FAQ page uses FAQPage schema with full question and answer pairs, making it directly citable by AI summaries and eligible for Google's rich result formats
  • All content is in the HTML source on first load, with nothing hidden behind JavaScript execution that a crawler might not wait for
  • Canonical URLs, Open Graph tags, and Twitter Card metadata are present on every page, giving every entry point a clean, consistent identity

The practical effect is that when someone asks an AI assistant about freelance web designers in Norfolk, or about bespoke hand-coded websites, this site is in a strong position to be cited, not because it paid for placement, but because it is genuinely readable and its content is well-structured and clearly attributed.

The outcome

The site is the proof. It loads fast, reads cleanly, works on every device, and holds up to scrutiny from both human visitors and machine crawlers. It ranks on its own terms.

More importantly, it makes a case. Clients who are serious about their digital presence, who understand that a website is not just a brochure but a system that works for the business, can look at this site and see what that actually means in practice.

WordPress remains the right answer for the majority of projects. Content-managed, flexible and maintainable by a non-technical team: the case for it is strong and real. But for a portfolio site, a high-performance brochure, or any project where technical excellence, search visibility, and AI discoverability are the priority, a bespoke hand-coded build is the better tool.

This site exists to show what that looks like.
